Le lecteur multimédia VLC est un lecteur, un encodeur et un diffuseur multimédia libre et gratuit fait par les volontaires de la communauté VideoLAN.
Ce super lecteur propose aussi des interfaces HTTP gratuites et excellentes pour Mobile Safari, Android et Web OS sur les Stores associés.

Versions

  • VLC Média Player 2.1.5 Rincewind.
  • Sarah 3.1.6
Mise à jour vers la version 4.0 de Sarah lorsqu'elle sera en version "Released".

vlcPlayer en quelques mots

  • Le contrôle par Sarah de vos médias audios, vidéos, flux radios, flux TV et photos.
  • Un mode VLC HTTP "Normal" avec interface et un mode HTTP sans interface.
  • Un contrôle des principales commandes du lecteur VLC.
  • La lecture directe d'un média.
  • Une base de données des médias par Types, Artiste/Catégorie, Album/Genre et Titre/Nom.
  • Un mode recherche simple et intuitif pour Sarah qui propose elle-même les titres en fonctions des critères que vous avez définis.
  • La modification simple de tous les thèmes et noms de média pour une prononciation "à l'Anglaise".
  • Plug-in multi-langages, tous les messages vocaux sont localisés.

1. Installation et configuration de VLC

Le lecteur VLC ne nécessite aucune configuration pour être utilisé avec vlcPlayer. Les requis pour une utilisation HTTP que vous pouvez trouver dans les documentations de VLC ne sont pas à exécuter. Toutes les configurations sont à réaliser du coté vlcPlayer.

4. Le classement et sauvegarde des informations des Médias

Le classement des médias est automatique et se fait par:
  • Le type de média.
  • Les noms de répertoires où sont sauvegardées les listes de lecture.
  • Les noms des listes de lecture.
Une base de données nedb (en mémoire) sauvegarde les informations extraites pour chaque média dans l'ordre des clés suivantes:
Clé
Description
Type
Type de Média.
Cette information est automatiquement récupérée du dernier répertoire de l'arborescence définie pour le type de média dans la propriété media_path du fichier vlcPlayer.prop.

Un exemple avec la propriété music_path:
C:/Mes Médias/Musiques
Tous les sous-répertoires de "Musiques" seront vu par vlcPlayer comme des listes de lecture de Type "Musiques".
Une recherche sur le type "Musiques" se fera sur les données référencées pour ce Type uniquement.

A noter qu'il est possible de mettre directement des listes de lecture sous le répertoire "Type".
Comme ces listes n'auront pas de catégorie, elle ne pourront être recherchées que par une recherche par "Titre"/"Nom".
Category
Catégorie du type de média.
Cette information est automatiquement récupérée du premier sous-répertoire du "Type".

Prenons un exemple avec un Média de type "Musiques" et un classement par "Artiste" avec 2 Artistes.
Mon arborescence se décompose ainsi:
C:/Mes Médias/Musiques/SuperTramp
                                               Pink Floyd

Les 2 "Artistes" sont "SuperTramp" et "Pink Floyd"

Lors d'une recherche par "Artiste" (nom donné pour une recherche de type musique), tous les médias à partir de "Category" seront listés

A noter qu'il est possible de mettre directement des listes de lecture sous un répertoire "Category".
Comme ces listes n'auront pas d'autres critères, elle ne pourront être recherchées que par une recherche par "Titre"/"Nom" et par "Artiste"/"Catégorie".
Genre
Genre de Categorie du type de média.
Cette information est automatiquement récupérée de tous les sous-répertoires d'une "Catégorie".

Reprenons notre exemple avec nos 2 Artistes préférés, j'y ajoute 2 albums pour chacun d'eux.
Mon arborescence se décompose ainsi:
C:/Mes Médias/Musiques/SuperTramp/Breakfast in America
                                                                       Crime of the Century
                                                   Pink Floyd/The Wall
                                                                       Atom Heart Mother

Les 2 "Albums" pour SuperTramp sont "Breakfast in America" et "Crime of the Century"
Les 2 "Albums" pour Pink Floyd sont "The Wall" et "Atom Heart Mother"

Lors d'une recherche par "Albums" (nom donné pour une recherche de type musique), tous les médias à partir de "Albums" seront listés

Les listes pourront être recherchées par "Titre"/"Nom", par "Artiste"/"Catégorie" et "Album"/"Genre".

Un peu plus loin...
Une information suplémentaire peut-être ajoutée dans la catégorisation et la recherche d'un média.
Vous pouvez ajouter des sous-répertoires à "Genre". Ceux-ci seront vu par vlcPlayer (et nommé par Sarah) comme des "Sections".
Nom
Nom d'une liste de lecture.
Cette information est automatiquement récupérée de tous les noms de listes de lecture trouvées dans tous les répertoires.

Plusieurs listes de lecture peuvent être sauvegardées dans chaque répertoire. Reprenons notre exemple avec nos 2 Artistes préférés, leurs 2 albums pour chacun d'eux, j'ajoute pour le 1er, 2 listes de lecture.
Mon arborescence se décompose ainsi:
.../Musiques/SuperTramp/Breakfast in America/Album complet.xspf
                                                                                     Le meilleur de Breakfast in America.xspf

Vous aurez compris, je pourrais donc lire soit l'album complet ou seulement les meilleurs titres que j'ai regroupés dans Le meilleur de Breakfast in America
Les déclinaisons sont nombreuses et peuvent être ajustées à votre convenance et pour chaque type de média.
Par exemple, la catégorie principale peut être un genre de musique, pop, rock, ou encore une année, etc... Il suffit juste de nommer correctement les répertoires et les play lists. Donc, pour les autres types de média, à vous de jouer!

Tous les noms de répertoires et listes de lecture peuvent être "anglisés" pour une prononciation plus correcte par Sarah. Cette personnalisation est effectuée dans le même fichier que pour la localisation de vlcPlayer situé dans vlcPlayer/lib/lang/PropValue.js. (voir la propriété "language")

"L'anglisation" est à réaliser dans le tableau "natural_voice" et par paires "key":"value".
Prenons un exemple avec une artiste et un album que Sarah, en bonne Française qui ne sait pas parler anglais, prononce comme une vache espagnole:
  • Bird York et son album, Seven Pound.
Dans le tableau "natural_voice", je vais ajouter:
  • "Bird York": "beurde yorque",
  • "Seven Pounds": "Séveune Pound",
Après cà, Sarah prononcera assez bien mon artiste préférée.
Faites, attention aux virgules en fin de paire.

5. La lecture directe d'une play list

Vous pouvez ajouter dans le fichier vlcPlayer.xml des commandes vocales directes pour ajouter à VLC une liste de lecture.

La clé out.action.command se décline comme ci-dessous:
  • "set_music" pour un média musique.
  • "set_video" pour un média vidéo.
  • "set_radio" pour un média flux radio.
  • "set_TVchannel" pour un média flux TV.
  • "set_picture" pour un média photo.
La clé out.action.key permet de cibler la play list associée à la commande vocale, elle se décline comme ci-dessous:
Valeur
Description
Artist ou Category
Category Media
Clé définie par paire type:value.
Cette clé est différenciée uniquement pour que Sarah formate son discour, soit:
  • "Artist" pour un média musique.
  • "Category" pour les autres médias.
Précisez l'artiste ou la catégorie de la play list, exemple:
Artist:SuperTramp

Si "out.action.key" ne contient que cette clé, toutes les play lists de l'artiste (ou la catégorie) seront ajoutées comme listes de lecture.
Album ou Genre
Genre du Media
Clé définie par paire type:value.
Cette clé est différenciée uniquement pour que Sarah formate son discour, soit:
  • "Album" pour un média musique.
  • "Genre" pour les autres médias.
Précisez l'album ou le genre, exemple pour un album de l'artiste SuperTramp:
Artist:SuperTramp,Album:Breakfast in America

Si "out.action.key" ne contient que l'Artist et l'Album, toutes les play lists de l'album(ou le genre) seront ajoutées comme listes de lecture.
Title ou Name
Nom du Media
Clé définie par paire type:value.
Cette clé est différenciée uniquement pour que Sarah formate son discour, soit:
  • "Title" pour un média musique.
  • "Name" pour les autres médias.
Précisez le nom de la play list , exemple pour un album de l'artiste SuperTramp:
Artist:SuperTramp,Album:Breakfast in America,Title:Le meilleur de Breakfast in America
Les valeurs de la clé de recherche de "key" sont séparées par des virgules.
Suivants la position de la liste de lecture, la clé "key" peut avoir différentes formes.

Exemple 1: Une play list "Mes Préférées" directement sous le Type Musiques (e.g. Chapitre 3)(pas d'Artist, ni d'Album). La clé key ne peut logiquement avoir que le "Title". 
‹item›musique Mes Préférées
‹tag›out.action.key="Title:Mes Préférées";out.action.command="set_music"‹/tag›‹/item›
Exemple 2: Toutes les play lists de l'album "Panique Celtique" de l'Artiste "Manau": 
‹item›musique Manau
‹tag›out.action.key="Artist:Manau,Album:Panique celtique";out.action.command="set_music"‹/tag›‹/item›
Exemple 3: La play list "Le meilleur de Tribute" de l'album "Tribute" de l'Artiste "John Newman": 
‹item›musique Le meilleur de Tribute
‹tag›out.action.key="Artist:John Newman,Album:Tribute,Title:Le meilleur de Tribute";out.action.command="set_music"‹/tag›‹/item›
Vous avez les explications pour faire vous-mêmes les commandes vocales directes pour les autres médias. :-)

6. La Recherche d'une play list par Sarah

Nous connaissons tous la difficulté qu'a Sarah à comprendre des termes qui ne sont pas dans son dictionnaire.
Pour cette raison et donc pour éviter de s'époumonner à répéter 10 fois un titre en montant dans les aigus sans résultat, un mode recherche permet de cibler une play list assez rapidement.

C'est donc la gentille Sarah qui va nous dire les titres disponibles:
  • Par Type de média
  • Par recherche typée
  • Dans l'ordre
  • La direction
que JE veux. Non mais, vilaine fille... :-D

6.1 Examinons d'abord les recherches typées

vlcPlayer propose 3 recherches typées:
  • Par Artiste ou Catégorie
  • Par Album ou Genre
  • Par Titre ou Nom
Mémo: Rappelez-vous, Artiste, Album et Titre pour la musique, le reste pour les autres médias.
Type
Description
Artiste ou Catégorie
Recherche par Artiste ou catégorie
Sarah effectue un ciblage des play lists qui ont une catégorie dans le Type défini dans la clé out.action.command du fichier vlcPlayer.xml
Album ou Genre
Recherche par Album ou Genre
Sarah effectue un ciblage des play lists des Albums ou Genre dans le Type défini dans la clé out.action.command du fichier vlcPlayer.xml
Titre ou Nom
Recherche par Titre ou Nom
Sarah effectue un ciblage des play lists par Titre ou Nom dans le Type défini dans la clé out.action.command du fichier vlcPlayer.xml

Note: En théorie, toutes les play lists.

6.2 L'ordre

L'ordre ne peut pas être changé et est défini par ordre alphabétique sur la première lettre du Type de recherche (tableau 5.1 çi-dessus).

Tout d'abord, lorsque une action de recherche est effectuée, le lazy lazyvlcSearch.xml est activé comme context.
Après un choix, on en sort automatiquement. Pas la peine d'utiliser l'action pour sortir qui est là uniquement pour annuler le mode (au cas où).

L'ordre alphabétique est automatique
  • par le début
  • par la fin
bien sûr, puisque par le début la recherche s'effectue de A à Z et par la fin de Z à A.

Mais (et c'est là que c'est très intéressant), suivant la commande vocale et la valeur de la clé out.action.pos, Sarah commence à rechercher à partir de la position de la lettre que vous avez défini. Vous trouverez dans le lazyvlcSearch.xml un exemple de début de recherche avec les commandes vocales:
  • par le milieu et plus
  • par le milieu et moins
qui, comme la commande le suggère, fait commencer la recherche alphabétique par Sarah à la position 13 qui est la position du M dans l'alphabet.
Il ne reste que l'imagination pour définir des commandes vocales et cibler des lettres qui vous interessent.

6.3 La direction

Le direction permet de rechercher par ordre alphabétique en montant ou descendant dans l'alphabet.
Autrement dit, de A à Z ou de Z à A.

Il est automatique pour la recherche par le début ou par la fin mais peut être configuré pour les autres recherches.
out.action.command="up" ou out.action.command="down"

6.4 Conclusion

En réfléchissant un tout petit peu pour savoir à quelle position dans l'alphabet la première lettre de la play list que l'on veut ajouter à VLC se situe, en ciblant le Type et la direction de recherche, la play list peut être retrouvée assez rapidement.

N'hésitez pas à ajouter des recherches dans le lazyvlcSearch.xml.

8. Configuration

vlcPlayer dispose d'un mode configuration vocale.
Lorsque la commande "mode configuration de vlc" est activée, le lazy lazyvlcConfig.xml est activé comme context.
Utilisez:
  • Merci sarah c'est tout Pour sortir du mode en sauvegardant en mémoire uniquement et sans changer la configuration dans le custom.prop de Sarah.
  • merci Sarah sauvegarde Pour sortir du mode et sauvegarder la configuration dans le custom.prop de Sarah.

Les configurations ou demandes possibles sont:
  • donne moi le status actuelListe l'état général de VLC
  • auto start à onPermet de configurer vocalement la propriété playlist_autostart à true du fichier vlcPlayer.prop
  • auto start à offPermet de configurer vocalement la propriété playlist_autostart à false du fichier vlcPlayer.prop
  • Met vlc en mode HTTPPermet de configurer vocalement la propriété mode à http du fichier vlcPlayer.prop
  • Met vlc en mode normalPermet de configurer vocalement la propriété mode à normal du fichier vlcPlayer.prop
